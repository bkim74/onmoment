# 온순간(OnMoment) MVP PWA — 통합 PRD v1.0

> "당신의 가장 연약한 순간이 누군가에게는 가장 강력한 선물이 됩니다."
> 최소 리소스 · 최대 효율 · 1인 개발 최적화

---

## 0. 문서 목적

7개 첨부문서(TRD v7.0, VC 리뷰 102개 우선순위, 리텐션 넛지 전략, refined 전략, 장소/공간 전략, 과기부 사업계획서, 에이전틱 AI 인프라)를 통합하여, **1인 개발자가 8주 내 출시 가능한 최소 기능 제품(MVP)** 기획서.

모든 기능은 **"이것이 없으면 서비스가 성립하지 않는가?"** 기준으로残酷히 잘라냄.

## 1. 제품 정의

### 1.1 한 줄 설명
**매일 1분 음성으로 40일간 자신의 이야기를 완성하고, 사랑하는 이에게 선물로 전달하는 시니어 정서 자산화 PWA**

### 1.2 해결하는 문제
| 문제 | 기존 해결책 | 온순간의 차이 |
|------|------------|--------------|
| 시니어의 삶이 기록되지 않음 | 일기장, 유튜브 | 1분 음성 = 진입장벽 최소 |
| 자녀와 깊은 대화 부재 | 전화, 카톡 | AI가 질문을 대신 생성 |
| 감정적 유산의 부재 | 사진첩, 상속 | 음성+AI가 서사를 완성 |

### 1.3 타겟 유저 (MVP는 1명에 집중)

**Primary: 기록자 (시니어 55-75세)**
- 스마트폰 기본 조작 가능 (카카오톡 사용 경험)
- 앱 설치 경험 적음 → PWA가 핵심
- "내 이야기를 남기고 싶다"는 내재적 동기

**Secondary: 수신자 (자녀 30-50대)** — v1.1에서 확장
- 부모님의 40일 여정 결과물을 수령
- 바이럴 루프의 시작점 (K-Factor 15% 목표)

## 2. 핵심 아키텍처: 불변의 파이프라인

모든 문서가 동일하게 검증한 **변하지 않는 코어**:

```
[1분 음성 녹음] → [Whisper STT] → [LLM 감정분석] → [UI 렌더링]
```

이 파이프라인은 B2C/B2B/어떤 여정 테마든 100% 동일. 프롬프트만 교체하여 완전히 다른 서비스로 작동 = **추가 개발 공수 0원**

## 3. 유저 여정 (5단계)

### Phase 1: 진입 — "선물로서의 프레이밍" (Day 0)
```
자녀가 링크 전달 → "엄마의 삶은 이미 가장 귀한 선물입니다"
→ 3일 미니 체험 리츄얼 (하이브리드 퍼널) → 40일 여정 선택
```
- **VC 리뷰 반영**: 가벼운 진입 → 깊은 여정으로 업셀링
- 3일 체험으로 즉각적 가치 체감 후, 40일 본 여정으로 전환

### Phase 2: 여정 — "매일 1분의 리츄얼" (Day 1-40)
```
알림 수신 → 내수용감각 프라이밍(30초) → 1분 음성 녹음
→ AI 구루 즉시 응답 → 감정 주파수 시각화
```
- **핵심 UX**: 2탭 이내 녹음 시작
- **감각적 UI**: 1-10점 슬라이더 대신 "오늘의 날씨" 아이콘 선택 (VC 피드백 반영)
- **4구루 동적 라우팅**: 감정 키워드 → 최적 AI 페르소나 자동 배정

### Phase 3: 응축 — "나만의 한 문장" (Day 38-40)
```
AI가 40일 데이터 분석 → 인생 키워드 추출 → 유저 최종 승인(Opt-in)
→ "나만의 한 문장" 확정
```
- Peak-End Rule 적용: 마지막 순간의 기억이 전체를 결정

### Phase 4: 선물 — "리츄얼 키트 전달" (Day 40+)
```
디지털 선물 페이지 생성 → 자녀에게 전달
→ NFC/QR 태그로 물건에 연결 (v1.1)
→ WebXR 공간 박제 (v2.0)
```

### Phase 5: 보존 — "디지털 성소" (지속)
```
선물 열람 → 음성 재생 → 장소 앵커링 → 가족 자산으로 영속
```

## 4. MVP 기능 명세 (v1.0)

### 4.1 반드시 들어가는 기능 (Must-Have)

| ID | 기능 | 설명 | 스펙 |
|----|------|------|------|
| F-01 | **음성 녹음** | 1분 음성 녹음 + 재녹음 | MediaRecorder API, 최대 90초 |
| F-02 | **STT 변환** | 음성→텍스트 실시간 변환 | OpenAI Whisper API |
| F-03 | **AI 감정 분석** | SDF 5레이어 분석 + 구루 응답 | Gemini Flash (비용 최소) |
| F-04 | **감각적 상태 입력** | 녹음 전 "오늘의 날씨" 아이콘 선택 | 5개 날씨 아이콘 (맑음~폭풍) |
| F-05 | **40일 타임라인** | 여정 진행 상황 시각화 | 진행률 바 + 마일스톤 카드 |
| F-06 | **마일스톤 보상** | 7/14/21일 정체성 카드 자동 생성 | AI 생성 요약카드 |
| F-07 | **한 문장 확정** | 40일 결과 AI 요약 → 유저 승인 | 편집 가능 텍스트 |
| F-08 | **선물 페이지** | 자녀 공유용 결과 페이지 | 정적 URL + 비밀번호 |
| F-09 | **알림** | 매일 녹음 리마인드 | 웹 푸시 알림 |
| F-10 | **안전망** | SDF 위험 감지 시 1393 연결 | 키워드 필터링 + 안내 |

### 4.2 v1.0에서 의도적으로 제외하는 기능

| 기능 | 제외 사유 | 예정 버전 |


## 6. 기술 스택 (최소 비용)

### 6.1 프론트엔드

| 항목 | 선택 | 이유 |
|------|------|------|
| 프레임워크 | **Next.js 14 (App Router)** | PWA 지원, SSR, SEO |
| PWA | **Serwist (next-pwa 계승)** | 오프라인 캐싱, 웹 푸시 |
| UI | **Tailwind CSS + shadcn/ui** | 시니어 친화 대형 UI, 빠른 개발 |
| 오디오 | **MediaRecorder API** | 네이티브 브라우저 녹음 |
| 상태관리 | **Zustand** | 경량, 단순 |

### 6.2 백엔드 / 인프라

| 항목 | 선택 | 이유 |
|------|------|------|
| BaaS | **Supabase** | 인증+DB+스토리지+Edge Functions |
| 인증 | **카카오 로그인 (Supabase Auth)** | 시니어 유일 친숙한 SSO |
| DB | **Supabase PostgreSQL** | RLS로 데이터 주권 보장 |
| 스토리지 | **Supabase Storage** | 음성 파일 (.webm) |
| Edge Functions | **Supabase Edge Functions** | STT/LLM API 호출 오케스트레이션 |

### 6.3 AI 파이프라인

| 단계 | API | 예상 비용/유저 |
|------|-----|----------------|
| STT | **OpenAI Whisper API** | ~₩600 (40회 × ₩15) |
| 감정분석+응답 | **Gemini 2.0 Flash** | ~₩2,000 (40회 × ₩50) |
| 마일스톤 요약 | **Gemini 2.0 Flash** | ~₩300 (4회) |
| 한 문장 생성 | **Gemini 2.0 Flash** | ~₩200 (1회) |
| **총 AI 비용** | | **~₩3,100/유저** |

### 6.4 인프라 비용 (월)

| 항목 | 비용 |

## 9. 리텐션 전략 (구조화 넛지)

### 9.1 일일 루프 (Day 1-40)

```
[푸시 알림] → "순자님, 오늘의 이야기를 들려주세요"
    ↓
[감각 프라이밍] → 날씨 아이콘 1탭 (내수용감각)
    ↓
[1분 녹음] → 탭 1회로 시작
    ↓
[즉시 보상] → AI 구루의 맞춤 응답 ("오늘 당신의 목소리에서 그리움을 읽었습니다")
    ↓
[진행 시각화] → 타임라인 업데이트 + % 증가
```

### 9.2 마일스톤 보상

| 일차 | 보상 | 효과 |
|------|------|------|
| Day 7 | **1주 정체성 카드** | "첫 번째 패턴이 보입니다" — 작은 승리 |
| Day 14 | **감정 지형도** | "지난 2주, 당신의 마음 날씨" — 시각화 |
| Day 21 | **중간 결산 편지** | "21일간 당신이 발견한 나" — 서사 응축 |
| Day 40 | **한 문장 + 선물** | "당신의 여정이 완성되었습니다" — 카타르시스 |

### 9.3 이탈 방어 (2-Day Rule)

- 연속 2일 미접속 시 푸시: "이틀째 비워진 페이지가 있어요"
- 3일째: 자녀에게 알림 (옵트인): "어머니의 여정이 잠시 쉬고 있어요"
- 7일째: "언제든 다시 시작할 수 있습니다" + 이전 기록 하이라이트

## 10. 안전망 프로토콜

### 10.1 SDF 위험 감지

```
음성 STT → 키워드 필터링 (자살/죽음/포기/슬픔)
  → 위험 감지 시:
    1. AI 응답 차단 (분석 결과 노출 금지)
    2. 따뜻한 안내 메시지: "지금 많이 힘드시군요"
    3. 1393 자살예방 핫라인 버튼 노출
    4. 응급 상황 시 119 연결 가이드
```

### 10.2 데이터 주권 (3중 분리)

- **Private**: 유저의 원본 기록 (본인만 열람)
- **Gift**: 유저가 Opt-in 승인한 선물 데이터 (자녀 열람)
- **Public**: 없음 (MVP에서는 퍼블릭 노출 없음)

## 13. 의사결정 기록 (Design Decisions)

| 결정 | 선택 | 이유 |
|------|------|------|
| 네이티브 앱 vs PWA | **PWA** | 시니어 앱 설치 장벽 제거, 1인 개발 비용 1/10 |
| 복수 SSO vs 카카오 only | **카카오 only** | 시니어 SSO 경험 = 카카오톡 90%+ |
| 1-10점 vs 날씨 아이콘 | **날씨 아이콘** | VC 피드백: 숫자 입력 인지 마찰 (우선순위 65) |
| 40일 vs 유연 기간 | **고정 40일** | 의미적 완결성 + 서사 구조 |
| 무료 vs 유료 | **유료 ₩29,000** | VC: 조기 WTP 검증 필수 (우선순위 2) |
| Gemini vs GPT | **Gemini Flash** | 비용 1/5, 한국어 품질 동급 |
| WebXR 포함 | **제외** | 8th Wall ₩수백만/월, MVP 불필요 |
| 보이스 클로닝 | **제외** | 유저당 $2-5, 마진 악화 |
